Critical Components

Security & Best Practices

มาตรฐานความปลอดภัยที่ต้องคำนึงถึงตั้งแต่เริ่มต้น (Secure by Design)

Identity & Access (การระบุตัวตน)

  • Auth: ใช้ OAuth 2.0 / OIDC และ Hashing รหัสผ่านด้วย bcrypt/Argon2
  • Authorization: ใช้ RBAC/ABAC และหลักการ Least Privilege
  • MFA: บังคับใช้การยืนยันตัวตนหลายปัจจัยสำหรับระบบสำคัญ

Vulnerability Prevention (การป้องกัน)

  • Injection: ป้องกัน SQL Injection ด้วย Prepared Statements/ORM
  • XSS: ทำ Input Validation และ Output Encoding เสมอ
  • CSP: ใช้ Content Security Policy จำกัดแหล่งที่มาของสคริปต์

Network & Session (เครือข่าย)

  • Transport: บังคับใช้ HTTPS/TLS และ HSTS ตลอดเวลา
  • Cookies: ตั้งค่า Secure, HttpOnly และ SameSite เพื่อกัน CSRF
  • CSRF: ใช้ Anti-forgery tokens สำหรับ State-changing requests

Ops & Monitoring (การดำเนินงาน)

  • Secrets: ห้าม Hardcode credential ใช้ Vault หรือ ENV variables
  • Scanning: ตรวจสอบ Dependencies อัตโนมัติ (Snyk, Dependabot)
  • Logging: บันทึก Audit logs และแจ้งเตือนเมื่อมีพฤติกรรมผิดปกติ
